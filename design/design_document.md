# تصميم التطبيق

## نوع التطبيق
بعد دراسة الخيارات المتاحة، تم اختيار تطبيق الويب (Web Application) للأسباب التالية:
- إمكانية الوصول من أي جهاز دون الحاجة للتثبيت
- سهولة التطوير والتحديث
- لا يحتاج إلى موافقات متاجر التطبيقات

## واجهة المستخدم
سيتكون التطبيق من الصفحات التالية:

### 1. الصفحة الرئيسية
- شعار التطبيق وعنوانه
- شرح موجز لوظيفة التطبيق
- زر "البدء" للانتقال إلى صفحة الموافقة

### 2. صفحة الموافقة
- شرح تفصيلي للبيانات التي سيتم الوصول إليها (جهات الاتصال والصور)
- توضيح أن البيانات سترسل إلى بوت تليجرام محدد
- زر "موافق" للمتابعة
- زر "إلغاء" للعودة إلى الصفحة الرئيسية

### 3. صفحة اختيار البيانات
- تبويبان: "جهات الاتصال" و"الصور"
- في تبويب جهات الاتصال:
  - قائمة بجهات الاتصال المتاحة مع خانات اختيار
  - زر "تحديد الكل" وزر "إلغاء تحديد الكل"
- في تبويب الصور:
  - عرض مصغر للصور المتاحة مع خانات اختيار
  - زر تحميل صور جديدة
- زر "التالي" للانتقال إلى صفحة المراجعة

### 4. صفحة المراجعة
- عرض البيانات المحددة (جهات الاتصال والصور)
- زر "تعديل الاختيار" للعودة إلى صفحة الاختيار
- زر "إرسال" لإرسال البيانات إلى بوت تليجرام

### 5. صفحة التأكيد
- رسالة تأكيد نجاح الإرسال
- زر "إرسال المزيد" للعودة إلى صفحة اختيار البيانات
- زر "العودة للرئيسية" للعودة إلى الصفحة الرئيسية

## تدفق البيانات

1. **طلب الإذن**:
   - عند النقر على زر "البدء"، يطلب التطبيق إذن الوصول إلى جهات الاتصال والصور
   - يتم عرض نافذة طلب الإذن القياسية للمتصفح

2. **اختيار البيانات**:
   - بعد منح الإذن، يقوم التطبيق بقراءة جهات الاتصال والصور المتاحة
   - يختار المستخدم البيانات التي يريد إرسالها

3. **مراجعة البيانات**:
   - يعرض التطبيق البيانات المحددة للمراجعة النهائية
   - يمكن للمستخدم تعديل اختياره قبل الإرسال

4. **إرسال البيانات**:
   - عند النقر على زر "إرسال"، يقوم التطبيق بإرسال البيانات إلى بوت تليجرام
   - يتم استخدام طلبات HTTP POST إلى واجهة برمجة تطبيقات تليجرام

## آلية الاتصال مع واجهة برمجة تطبيقات تليجرام

### إرسال الصور
```javascript
// لكل صورة محددة
async function sendPhoto(photo, chatId, botToken) {
  const formData = new FormData();
  formData.append('chat_id', chatId);
  formData.append('photo', photo);
  
  const response = await fetch(`https://api.telegram.org/bot${botToken}/sendPhoto`, {
    method: 'POST',
    body: formData
  });
  
  return await response.json();
}
```

### إرسال جهات الاتصال
```javascript
// لكل جهة اتصال محددة
async function sendContact(contact, chatId, botToken) {
  const params = {
    chat_id: chatId,
    phone_number: contact.phoneNumber,
    first_name: contact.firstName,
    last_name: contact.lastName || ''
  };
  
  const response = await fetch(`https://api.telegram.org/bot${botToken}/sendContact`, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify(params)
  });
  
  return await response.json();
}
```

## آليات الموافقة والخصوصية

1. **طلب الإذن الصريح**:
   - طلب إذن الوصول إلى جهات الاتصال باستخدام `navigator.contacts.select()`
   - طلب إذن الوصول إلى الصور باستخدام `<input type="file" accept="image/*" multiple>`

2. **سياسة الخصوصية**:
   - توضيح كيفية استخدام البيانات
   - التأكيد على عدم تخزين البيانات على الخادم
   - شرح أن البيانات ترسل مباشرة إلى بوت تليجرام المحدد

3. **مراجعة البيانات**:
   - عرض البيانات المحددة قبل الإرسال
   - إمكانية إلغاء أو تعديل الاختيار

4. **تأكيد الإرسال**:
   - طلب تأكيد نهائي قبل إرسال البيانات

## اعتبارات الأمان

1. **حماية بيانات المستخدم**:
   - عدم تخزين البيانات على الخادم
   - إرسال البيانات عبر HTTPS
   - حذف البيانات من الذاكرة بعد الإرسال

2. **التحقق من صحة البيانات**:
   - التحقق من صحة تنسيق أرقام الهواتف
   - التحقق من حجم الصور وتنسيقها

3. **حماية من هجمات الحقن**:
   - تنظيف البيانات المدخلة
   - استخدام معلمات محددة في طلبات API
